Digital Notice Board System - Features & AI Implementation Prompt

1.  Authentication Implementation Prompt: Create a secure authentication
    module with Admin, Staff, and Student roles. Include login, logout,
    password hashing, session management, forgot/reset password, and
    profile management.

2.  Dashboard Implementation Prompt: Build a dashboard showing total
    notices, active notices, expired notices, users, recent notices, and
    analytics cards.

3.  Notice Management Implementation Prompt: Implement CRUD for notices
    with title, description, category, priority, attachments, publish
    date, expiry date, target audience, archive, restore, duplicate,
    pin, and scheduling.

4.  Categories Implementation Prompt: Allow administrators to create,
    edit, and delete notice categories.

5.  Approval Workflow Implementation Prompt: Implement a workflow where
    staff submit notices, admins approve/reject them before publication.

6.  Target Audience Implementation Prompt: Allow notices to target
    everyone or specific faculties, departments, programmes, levels,
    staff, or students.

7.  Attachments Implementation Prompt: Support secure upload and
    preview/download of PDF, DOCX, images, and ZIP files.

8.  Search & Filters Implementation Prompt: Provide keyword search and
    filters by category, date, priority, status, and audience.

9.  Notifications Implementation Prompt: Implement in-app notifications
    and optional email alerts for new notices.

10. Expiry & Scheduling Implementation Prompt: Automatically publish
    scheduled notices and archive expired notices.

11. Priority & Pinning Implementation Prompt: Support High/Medium/Low
    priority and pinned notices that always appear first.

12. Read Tracking Implementation Prompt: Track notice views and display
    read/unread statistics.

13. Bookmarks Implementation Prompt: Allow users to bookmark notices for
    later viewing.

14. Analytics Implementation Prompt: Display charts for monthly notices,
    views, category distribution, and most viewed notices.

15. User Management Implementation Prompt: Allow admins to manage users,
    roles, activation, and departments.

16. Role-Based Access Implementation Prompt: Restrict features based on
    Admin, Staff, and Student permissions.

17. Calendar Implementation Prompt: Display upcoming events and notice
    deadlines in a calendar view.

18. Activity Logs Implementation Prompt: Log all important system
    actions with timestamps and users.

19. Reports Implementation Prompt: Generate PDF, Excel, and CSV reports
    for notices, users, and analytics.

20. Responsive UI Implementation Prompt: Create a responsive interface
    with light/dark mode and accessibility considerations.

21. Security Implementation Prompt: Implement input validation, CSRF
    protection, XSS prevention, SQL injection prevention, secure file
    uploads, and audit trails.

Recommended Database Tables: users, roles, faculties, departments,
programmes, levels, categories, notices, notice_attachments,
notice_views, bookmarks, notifications, activity_logs, archived_notices.
